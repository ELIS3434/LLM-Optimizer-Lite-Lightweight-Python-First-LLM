# Security Policy

## Supported versions
This is an early-stage project. Security fixes are applied on the latest mainline state.

## Reporting a vulnerability
Please do not open public issues for vulnerabilities.
Report privately with:
- impact summary
- reproduction steps
- affected files or endpoints

## Operational recommendations
- Use unique API keys in production.
- Set `API_KEY_ENABLED=true` in production.
- Rotate credentials regularly.
- Prefer PostgreSQL for production workloads.
